ANIMATING SPRITES

To begin adding animation to our character sprites, we have to create two child nodes underneath the character body node we created previously. The two new child nodes should be called AnimationTree and AnimationPlayer.

AnimationPlayer allows us to create keyframes in order to determine what the character sprite is doing. This can be done by creating a new property track and assigning it to the Sprite2D node we created previously. We can select the frames of the Sprite2D nodes that we want associated with each action, and determine the time how frequently it switches between each of the keyframes. In my case, I had the keyframes switch every 0.2 sections, and having each action loop after 0.4 sections.

Once we are satisfied with how we want to animation to appear, we would move onto the AnimationTree node. The AnimationTree node tells Godot how we want each action to blend into one another by creating a tree of each action's relationship with one another.

Because I want the idle animation to play when the user is not moving, and the walk cycle to play when the user is moving, I created a tree that starts with the root node, then pointing to the idel node, which points at the walk node. This walk node then points back to the idle node, creating a cycle.

You will see that next to both the walk and idle nodes, there is a little pencil icon, which is what you can click on to edit the node. Within the node, we can assign an action to user movement based on an x and y coordinate system.

When all of these steps are completed, you should be able to run the program and see the character sprite move accordingly your keypresses.
